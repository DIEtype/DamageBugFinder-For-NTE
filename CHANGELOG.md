## 0.5.2 - 2026-07-20

- 桌面版改为直接加载计算器，不再通过受限 iframe 和消息桥转发桌面 API。
- 移除构建工具自动加入的在线图标脚本，避免离线环境等待网络导致页面迟迟无法完成加载。
- 桌面环境直接显示 OCR 入口，不再依赖异步桥接成功后才出现。
- WebView2 默认使用软件兼容渲染，规避游戏、显卡叠加层或驱动状态引起的 GPU 初始化阻塞。
- 新增桌面页面结构与 OCR 入口的启动自检。

## 0.5.1 - 2026-07-20

- 新增无需每次解压运行环境的快速启动目录版，推荐日常测试使用。
- OCR 与图像组件改为点击“自动读取游戏面板”后才加载，降低普通启动负担。
- 当前数据已经处于待验算状态时，不再随每次键盘输入重复清空和重排结果区。
- 保留单文件版作为便携备选，并在 README 中说明两种封装的启动差异。

## 0.5.0 - 2026-07-19

- 桌面版加入游戏窗口自动捕获，无需手动保存或选择日常测试截图。
- 截图倒计时期间自动隐藏计算器并切回游戏，允许打开或滚动属性详情。
- 使用 Windows 本机简体中文 OCR，识别过程完全离线且不保存完整截图。
- 支持多张滚动面板合并、重复值交叉验证、冲突提示和确认后填表。
- 自动填写总攻击、基础攻击、角色等级、暴击伤害、通用/属性增伤、环合强度与治疗加成。
- 七种属性伤害增强统一映射到属性增伤，并排除属性伤害抗性行。
- 保留多图手动导入作为备用入口，HTML 版继续保持纯静态计算功能。
- 桌面 EXE 自检新增离线 OCR 回归入口。

## 0.4.1 - 2026-07-19

- 将非战斗面板环合强度限制为 `0～360`。
- 觉醒、武器等百分比和固定提升仍可使最终环合强度超过 360。
- 载入旧配置时自动把超出范围的基础环合强度收敛到合法范围。

## 0.4.0 - 2026-07-19

- 将伤害计算与治疗计算改为独立界面模式，自动隐藏无关输入区。
- 治疗公式改为“战斗面板攻击 × 治疗倍率 ×（1 + 治疗加成）”。
- 加入基础治疗加成及觉醒、武器等候选治疗加成的生效/失效排查。
- 加入环合强度、百分比提升和固定数值提升。
- 加入仅作用于常规伤害的浸染乘区与可选特殊独立乘区。
- 创生花公式改为“9000 × 防御区 × 抗性区 ×（1 + 环合强度 / 600）”。
- 明确浸染和特殊独立乘区不作用于创生花。
- 更新 README 的游戏内字段位置、敌人抗性数据来源、公式说明和 Git 上传清单。
- 加入 MIT License。

## 0.3.0 - 2026-07-19

- 加入标准、深度和极限三档搜索强度。

- 重写 README，加入实机截图、逐字段填写规则、完整结果解释和性能边界。
