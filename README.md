# 游戏数值验算器

一个完全离线的游戏数值测试工具，用于正向计算伤害与治疗、验证觉醒和武器效果是否生效，并从大量未知伤害中反推可能来源。仓库同时提供 Windows 桌面 EXE 和单文件 HTML。

## 直接使用

Windows 用户优先双击：

```text
dist/GameDamageCalculator.exe
```

EXE 不需要安装 Python，也不会打开外部浏览器。程序使用 Windows 10/11 通常自带的 Microsoft Edge WebView2 Runtime。角色、版本和草稿保存在 `%LOCALAPPDATA%\GameDamageCalculator`。

也可以直接双击 `index.html` 使用网页单文件版。网页版本的数据由打开它的浏览器保存。

也可以将整个目录部署到任意静态托管服务；入口文件为仓库根目录的 `index.html`。

## 主要功能

- 计算常规技能的非暴击与暴击伤害。
- 计算固定基础值为 9000 的创生伤害。
- 独立计算治疗技能。
- 支持多个伤害技能与治疗技能。
- 支持技能倍率加算和乘算。
- 按角色名称和游戏版本保存配置。
- 自动记忆尚未保存的当前草稿。
- 一键清空当前输入和草稿，同时保留已保存的角色/版本档案。
- 枚举基础乘区、觉醒和武器效果的生效或失效状态。
- 检测疑似误用其他技能倍率的情况。
- 将误差限制在每条实测值 ±20 以内。
- 支持“狂暴溯源”，批量识别大量未知伤害的可能来源。

## 两种工作模式

### 常规验算

适用于已经知道伤害来自哪个技能的情况。为每个技能填写倍率、暴击状态和实测伤害，工具会联合判断：

- 哪些觉醒或武器提升生效；
- 哪些基础乘区可能失效；
- 技能是否疑似读取了其他技能的倍率；
- 哪些组合的理论显示值最接近实测值。

### 狂暴溯源

适用于同时出现大量伤害、无法判断来源的情况。将伤害数字用空格、逗号或换行批量粘贴，工具会为每条数据尝试：

- 所有已启用的常规技能；
- 创生伤害；
- 暴击和非暴击；
- 本技能倍率和其他已记录技能倍率；
- 所有可能的基础乘区及觉醒、武器生效状态。

同一来源允许重复出现。±20 内没有候选时，该条数据只会标记为“无来源”，不会强行匹配到差距很大的结果。

狂暴溯源目前假设同一批伤害共享同一套角色面板、敌人面板和全局增益状态。

## 如何理解验算结果

“最符合实测的全局状态”不是某一条伤害的来源，而是整批实测值共同使用的一套假设。例如：基础防御穿透是否生效、某项觉醒是否生效、某项武器增伤是否生效。程序用同一个全局状态重新计算本批全部数据，再对所有状态排序。

每条数据的技能、暴击状态和倍率来源在“最佳全局状态的逐项验算”中单独显示。

- `误差`：`|理论显示值 - 实测值|`。理论精确值会先按游戏显示规则四舍五入。
- `±20 内来源数`：在当前全局状态下，这条未知伤害有多少个“技能 × 暴击状态 × 倍率来源”候选落在 ±20 内。它不是效果组合数量。
- `显示值总误差`：常规验算为本批全部条目的误差之和；狂暴溯源只累加已经在 ±20 内匹配成功的条目。
- `零误差全局状态（最多 3 项）`：只列出整批显示值总误差为 0 且没有缺失来源的生效/失效假设。存在多个时，第一项是解释优先级最高的状态，后面是备选状态。

状态排序依次优先：无来源或超限条目更少、相对误差总和更小、基础乘区失效更少、误用其他技能倍率更少、觉醒和武器效果失效更少。这是确定性的解释优先级，不是统计学概率。

如果不存在零误差状态，顶部仍会显示总体最接近的状态用于排查，但右侧备选列表不会显示带误差的近似状态。

## 搜索策略

- 判断项不超过 18 个时完整枚举，不遗漏排列。
- 判断项超过 18 个时，从“全部生效”状态开始逐项排除。
- 大规模搜索保留 5000 条路径，并显示已评估状态数与理论总状态数。
- 同误差时优先基础乘区正常、觉醒和武器少失效、本技能倍率正常的解释。

限宽搜索能够显著降低大规模组合的计算时间，但不能保证覆盖全部理论排列。

## 数据保存

角色和版本配置保存在本机，不会上传到网络，也不会写入 Git 仓库。桌面版使用固定的 `%LOCALAPPDATA%\GameDamageCalculator`；网页版本使用浏览器本机存储。

删除桌面版应用数据目录，或清理网页版本的浏览器站点数据后，已保存配置会丢失。重要配置建议同时保留原始测试表格。

## 重新构建 EXE

需要 Windows、Python 3.13 以及可联网安装依赖的环境。在仓库根目录运行：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r .\desktop\requirements-build.txt
powershell -NoProfile -ExecutionPolicy Bypass -File .\desktop\build.ps1
```

输出文件为 `dist/GameDamageCalculator.exe`。每次修改 `index.html` 后重新执行最后一条命令即可。
